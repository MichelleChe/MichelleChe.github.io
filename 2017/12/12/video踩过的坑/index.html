video的坑

发表于 2017-12-12 | 分类于 js

今天突然发现在Mac的Safari中在http下video就不能自动播放，报错

一个令人费解的错误，但是在本地能够播放，经过百度之后是新版的Safari禁止了video和audio的自动播放，必须用户手动触发才能进行播放，就和在移动端一样，但是可以通过给video添加一个muted的属性让他自动播放。

还有一个在Safari中的坑就是当你使用rotate旋转的时候，盒子的z-index会不起作用，这个时候可以给他添加transform：translateZ(100)如果不够就给大一点，直到达到你想要的效果即可，还有个方法就是在外层添加overflow：hidden也是可以的。

当然我不是用这两个方法解决的，我直接就丢弃了制造bug的源头，没有用rotate而是使用scale来代替我想要的旋转的效果，这下就不担心层级的问题，或者外层的问题了。如果你的要旋转的物体是左右对称的就可以采用scale来代替rotate，是一样的效果,代码如下

video在移动端是最令我头疼的，特别是在iOS手机中，当然有video.js你可以用这个插件，但是在移动端多引入一个js未免显得很多余。

在iOS手机中的QQ浏览器video会有一个小窗口，没有办法控制，比如这种我已经关闭了视频弹窗但是视频仍然在那里必须全屏一下再退出才能关掉这个小窗口。其实这个很好解决就是video全屏就解决这个问题了。

比如你的代码结构是这样的

<div class="pop pop_video">
  <div class="pop_close"></div>
  <video class="video" controls="">
    <source src="/videos/Sdorica_2.mp4" type="video/mp4">
    您的浏览器不支持 video 标签。
  </video>
</div>

样式这么写就可以了

.pop{
      position: relative;
    top: 0;
    left: 0;
    right: 0;
    min-height: 100vh;
    overflow: auto;
    text-align: center;
    background: rgba(0,0,0,.8);
    z-index: 99;
}
.pop_video video {
    width: 100%;
    position: absolute;
    top: 50%;
    left: 50%;
    -webkit-transform: translate(-50%,-50%);
    -moz-transform: translate(-50%,-50%);
    -ms-transform: translate(-50%,-50%);
    -o-transform: translate(-50%,-50%);
    transform: translate(-50%,-50%);
}

这样子就完全避免的那个小窗口的尴尬问题，点进去播放直接就是小窗口放大的效果。

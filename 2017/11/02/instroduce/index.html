hexo的next的主题的坑

发表于 2017-11-02 | 分类于 hexo

第一次用hexo搭建自己的博客，看见网上都用的next的主题，为了更加的美观的体验我也用了next这个主题，总结在这个过程中遇见的问题

主要参考这篇文章免费个人博客搭建http://www.jianshu.com/p/380290deb8f0作者写的很详细了，我在这里不再多加赘述，可以参考这个来搭建你的个人博客。这里主要说一下那篇博客没有说的遇见的坑。
1、当你安装完git或者node或者hexo等等之后一定记得配置环境变量，否则在全局中是不认识那个命令的。
2、当你什么都配置好以后，开始优美化你的主题的时候记得是两个配置文件，一个是hexo的还有一个就是设置主题的各种的配置，切记这是两个文件一个在根目录下的_config.yml另一个是在\themes\next_config.yml下的文件，当时就是没注意改了半天没反应
3、在配置文件的冒号后面有个空格，如果没有的话就会是一片空白。
4、写博客的时候想要添加图片的方法是下载一个插件来加载本地图片使用方法

a.首先确认_config.yml 中有 post_asset_folder:true。设置post_asset_folder为true参数后，在建立文件时，Hexo
会自动建立一个与文章同名的文件夹，您可以把与该文章相关的所有资源都放到那个文件夹。
b.然后npm install hexo-asset-image –save
c.比如新建一个项目结构目录如
MacGesture2-Publish
├── apppicker.jpg
├── logo.jpg
└── rules.jpg
MacGesture2-Publish.md
然后使用![logo](MacGesture2-Publish/logo.jpg) 就可以插入图片。其中[]里面不写文字则没有图片标题。

5、博客写好之后发布

如果是第一次部署，终端会提示要求输入用户名和密码。等命令执行完之后，过几分钟打开 http://xxx.github.io 即可看到你的个人博客了。以后要发布新文章，执行上述命令即可。

css规范

发表于 2017-11-02 | 分类于 css

上周看了一篇博客关于jQuery源码解读的，我觉得博主理解和分析的让阅读的人很容易的明白，然后继续看了看他的博客，一篇介绍css规范的文章，我觉得不论是在团队合作中还是考虑到浏览器的性能，css规范还是很有必要的，我曾经书写css就是想到一条是一条直接就在后面书写，完全不考虑reflow和repaint，知道有一天我去面试的时候面试官问我reflow是什么的时候，什么样的情况下会导致reflow和repaint才知道css的重要性。

属性的顺序

1、有content属性的时候应该放在最前面，比如用伪类before，after的时候
2、布局方式：比如position、left、right、top、bottom、z-index、display、float
3、盒模型：比如width、height、padding、margin、border、overflow…
4、文本排版：比如line-height、font、text-align、word-wrap…
5、视觉外观: 比如color、background、transform、transition、list-style、animation…
布局方式 处在第一位，因为他可以使一个元素脱离正常文本流，并且覆盖盒模型相关的样式。盒模型紧跟其后，因为他决定了一个组件的大小和位置。其他属性只在组件内部起作用或者不会对前面两种情况的结果产生影响，所以他们排在后面。
